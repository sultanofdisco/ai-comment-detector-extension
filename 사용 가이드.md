# X AI 댓글 탐지기 사용 가이드

## 1. 개요

이 익스텐션은 X(트위터) 페이지에서 댓글을 읽어서 로컬 분석 서버로 보내고, 댓글 옆에 AI 의심 배지를 붙여주는 크롬 확장 프로그램입니다.

현재 동작 흐름은 다음과 같습니다.

1. X 댓글 페이지에 접속한다.
2. 익스텐션의 content script가 댓글 텍스트와 작성자 정보를 추출한다.
3. background service worker가 로컬 서버 `http://127.0.0.1:8000`의 `/predict` API를 호출한다.
4. 서버가 1차 `human/ai` 판별을 하고, 필요하면 2차 LLM 분류를 수행한다.
5. 익스텐션이 댓글 옆 배지를 표시하고, 결과를 side panel에도 반영한다.

---

## 2. 준비물

- Google Chrome
- 로컬 분석 서버
- 이 익스텐션 프로젝트 폴더

현재 서버 주소는 아래로 고정되어 있습니다.

```text
http://127.0.0.1:8000
```

서버 주소를 바꿔야 하면 아래 파일의 `SERVER_BASE_URL` 값을 수정해야 합니다.

- `background/service-worker.js`

---

## 3. 서버 먼저 실행하기

익스텐션은 단독으로 동작하지 않고, 반드시 분석 서버가 먼저 켜져 있어야 합니다.

### 서버 실행 예시

```powershell
cd C:\Users\user\ai-comment-detector-server
.\.venv\Scripts\Activate.ps1
python -m uvicorn server.app:app --reload
```

### 서버 상태 확인

브라우저나 터미널에서 아래 주소가 열리면 정상입니다.

```text
http://127.0.0.1:8000/health
```

정상 응답 예시:

```json
{
  "status": "ok",
  "stage1_model_dir": "...",
  "stage2_model_dir": "...",
  "device": "cpu",
  "ai_threshold": 0.85
}
```

## 4. 크롬에 익스텐션 로드하기

### 1. 확장 프로그램 페이지 열기

크롬 주소창에 아래를 입력합니다.

```text
chrome://extensions
```

### 2. 개발자 모드 켜기

오른쪽 위의 `개발자 모드`를 켭니다.

- `압축해제된 확장 프로그램을 로드합니다` 클릭
- 아래 폴더 선택

```text
C:\Users\user\ai-comment-detector-extension
```

### 4. 로드 후 확인

- 확장 프로그램 이름이 목록에 표시되는지 확인
- 오류가 뜨지 않는지 확인
- 필요하면 `새로고침` 버튼으로 다시 로드

## 5. 실제 사용 방법

### 1. X 댓글 페이지 열기

아래 중 하나의 페이지로 이동합니다.

- `https://x.com/...`
- `https://twitter.com/...`

댓글이 포함된 페이지여야 합니다.

### 2. 댓글 자동 분석 확인

페이지가 열리면 익스텐션이 `article` 요소를 스캔해서 댓글을 자동 분석합니다.

결과는 댓글 옆에 다음처럼 표시됩니다.

- `AI 의심`
- `AI 의심 / GPT 추정`
- `AI 의심 / Claude 추정`
- `AI 의심 / Gemini 추정`
- `AI 의심 / DeepSeek 추정`

`human`으로 분류된 댓글은 배지가 표시되지 않을 수 있습니다.

### 3. side panel 확인

분석 결과는 크롬 side panel에서도 볼 수 있습니다.

side panel에서는 다음 정보를 확인할 수 있습니다.

- 최종 라벨
- confidence
- ai_score
- top2 후보
- risk level
- 작성자 핸들

---

## 6. 결과 해석 방법

### `pred_label`

최종 예측 결과입니다.

- `human`
- `ai`
- `gpt`
- `claude`
- `gemini`
- `deepseek`

### `ai_score`

1차 모델이 계산한 AI 확률입니다.

- `0.5` 미만이면 보통 `human`
- `0.5` 이상이면 `ai` 가능성 있음
- `0.85` 이상이면 2차 모델이 LLM 종류를 추가 분류

### `confidence`

최종 라벨에 대한 confidence입니다.

### `risk_level`

서버가 계산한 위험 구간입니다.

- `low`
- `medium`
- `high`

---

## 7. 결과가 저장되는 방식

익스텐션은 분석 결과를 `chrome.storage.local`에 저장합니다.

같은 댓글을 다시 만났을 때는 서버를 다시 호출하지 않고 캐시된 결과를 재사용합니다.

즉,

- 처음 분석할 때는 서버 요청 발생
- 이후 같은 댓글은 storage 캐시 사용

---

## 8. 다시 테스트할 때

기존 캐시 때문에 예전 결과가 그대로 보일 수 있습니다.

그럴 때는 아래 중 하나를 하면 됩니다.

### 방법 1. 익스텐션 새로고침

- `chrome://extensions`
- 해당 익스텐션의 새로고침 버튼 클릭

### 방법 2. 저장소 비우기

확장 프로그램의 storage를 비우고 다시 테스트합니다.

### 방법 3. 페이지 새로고침

X 페이지를 새로고침해서 content script를 다시 실행합니다.

## 9. 문제 해결

### 1. 배지가 안 뜰 때

아래를 순서대로 확인합니다.

- 서버가 켜져 있는지 확인
- `http://127.0.0.1:8000/health`가 열리는지 확인
- 익스텐션이 로드되어 있는지 확인
- X 페이지가 `x.com` 또는 `twitter.com`인지 확인
- 댓글 요소가 실제로 `article` 구조인지 확인

### 2. 서버 연결 실패가 날 때

확장 프로그램은 현재 `http://127.0.0.1:8000`으로 요청을 보냅니다.

즉, 아래를 확인합니다.

- FastAPI 서버 실행 여부
- 포트 `8000` 사용 여부
- 서버 주소를 바꿨다면 `background/service-worker.js` 수정 여부

### 3. side panel에 아무것도 안 뜰 때

- 댓글이 실제로 분석되었는지 확인
- `human` 댓글만 있는 경우 side panel이 비어 있을 수 있음
- 확장 프로그램 storage에 결과가 저장되었는지 확인

### 4. 모델이 없다고 나올 때

서버 쪽 `saved_models` 폴더가 비어 있으면 `/predict`가 실패할 수 있습니다.

다음을 확인합니다.

- `saved_models/stage1_kcbert_binary`
- `saved_models/stage2_llm_hybrid`

## 10. 개발 중 자주 쓰는 확인 포인트

### 익스텐션 background 로그 보기

- `chrome://extensions`
- 해당 확장 프로그램의 `서비스 워커` 클릭
- 콘솔에서 `/health`, `/predict` 요청 결과 확인

### content script 동작 확인

X 페이지에서 개발자 도구를 열고 콘솔 로그를 확인합니다.

### 서버 로그 확인

FastAPI가 실행 중인 터미널에서 `/predict` 요청이 들어오는지 봅니다.

## 11. 추천 사용 순서

1. 서버 실행
2. `/health` 응답 확인
3. 크롬에 익스텐션 로드
4. X 댓글 페이지 접속
5. 댓글 옆 배지 확인
6. side panel에서 상세 결과 확인

## 12. 현재 기준 파일 역할

- `manifest.json`
  - 확장 프로그램 설정
- `background/service-worker.js`
  - 서버 통신, 캐시 저장
- `content/content-script.js`
  - 댓글 추출, 배지 렌더링
- `ui/sidepanel.html`
  - side panel 화면
- `ui/sidepanel.js`
  - 저장된 결과 렌더링
